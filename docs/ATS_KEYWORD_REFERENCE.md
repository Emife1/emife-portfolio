## âš ï¸ ATS Do's and Don'ts

### âœ“ DO:
- Use standard section headers (Education, Experience, Skills, Certifications)
- Spell out acronyms first time: "SOX (Sarbanes-Oxley)"
- Use standard keywords matching job postings
- Bold/highlight key terms (helps human reviewers)
- Keep formatting simple (avoid graphics, columns, tables in raw ATS)
- Include quantified metrics: "$500K+", "50%", "8 projects"

### âœ— DON'T:
- Use graphics, columns, or complex formatting in ATS resume version
- Misspell system names (NetSuite, not "Net Suite")
- Use "unusual" fonts or excessive styling
- Stuff keywords unnaturally ("AP, AP, AP, AP enthusiast")
- Hide text in white text or tiny fonts
- Use inconsistent acronyms (AP vs Accounts Payable randomly)
- Avoid jargon that doesn't match job descriptions
